# Автоматизация тестирования Wikipedia (Web, Mobile, API)
Комплексный проект по разработке автоматизированных сценариев для проверки различных уровней системы Wikipedia: пользовательского интерфейса сайта, мобильного приложения и серверного API.
* Язык программирования: Java 21
* Тестовый фреймворк: TestNG
* Автоматизация Web: Selenium WebDriver 4.27.0
* Автоматизация Mobile: Appium 9.3.0 (драйвер UIAutomator2)
* Тестирование API: REST Assured 5.4.0
* Сборка и зависимости: Maven
* Логирование: SLF4J + Logback
Проект реализован с использованием паттерна Page Object Model (POM), что позволяет отделить логику взаимодействия с элементами от логики самих тестов.
* **BaseWebTest**: автоматизирует запуск ChromeDriver через WebDriverManager, настраивает таймауты и управляет закрытием браузера после тестов.
* **BaseMobileTest**: конфигурирует подключение к Appium Server, задает специфические возможности (Capabilities) для устройства Poco X7 и управляет установкой APK-файла.
**Методы WikipediaHomePage:**
* `enterSearchText(String text)`: выполняет ввод данных и имитирует нажатие клавиши ENTER для перехода к результатам.
* `clickSearch()`: обеспечивает безопасный клик по кнопке поиска.
* `isMainPageLinkDisplayed()`: верифицирует корректную загрузку интерфейса главной страницы.
**Методы WikipediaMobilePage:**
* `dismissPopups()`: критически важный метод, который в фоновом режиме проверяет наличие системных окон, Onboarding-экранов и игровых анонсов Wikipedia Alpha, закрывая их для обеспечения стабильности тестов.
* `getArticleTitle()`: считывает атрибут `content-desc` элемента заголовка для проверки точности поиска.
* `scrollToBottom()`: реализует прокрутку статьи с использованием Android UIAutomator.
Используется для проверки серверной логики (Rest V1 API). Проверяется корректность JSON-схемы ответа и обработка ошибок (код 404).
mvn clean test -Dtest=WikipediaApiTest
mvn clean test -Dtest=WikipediaWebTest
1. Запустите Appium: `appium`
2. Подключите Android-устройство (Отладка по USB включена).
3. Выполните:
mvn clean test -Dtest=WikipediaMobileTest
Для мониторинга процесса тестирования настроен вывод логов в консоль и файл `test-output.log`. Это позволяет отслеживать этапы инициализации драйверов, ввод данных и моменты срабатывания логики закрытия всплывающих окон.
